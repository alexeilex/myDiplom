## Что решает алгоритм

Метод сопряжённых градиентов (Conjugate Gradient, CG) применяется для решения линейной системы
$A x = b$, где матрица (A) должна быть **симметричной положительно определённой**.

В задаче Пуассона после дискретизации мы получаем именно такую систему:
$-\Delta u = f.$
После аппроксимации на сетке это превращается в большую разреженную систему, которую удобно решать итерационно.

---

## Почему matrix-free

В matrix-free варианте матрицу (A) не собирают явно. Вместо этого задают функцию, которая по вектору (v) возвращает произведение (A v).

Для Пуассона это удобно, потому что оператор Лапласа можно вычислять локально по сетке через шаблон конечных разностей.

---

## Идея метода

CG строит последовательность приближений (x^k), используя:

- невязку $(r^k = b - A x^k)$,
- направление поиска ($p^k$),
- шаг ($\alpha_k)$,
- коэффициент обновления $(\beta_k)$.
Метод не идёт просто по невязке, как [[Итерации Ричардсона]], а строит **сопряжённые направления**, чтобы не портить уже найденные компоненты решения.

---

## Алгоритм CG

Пусть задано начальное приближение (x^0).

1. Вычислить невязку:
$r^0 = b - A x^0.$ 
2. Положить $p^0 = r^0.$  
3. Для $(k = 0,1,2,\dots)$ повторять:

$\alpha_k = \frac{(r^k, r^k)}{(p^k, A p^k)}$

$x^{k+1} = x^k + \alpha_k p^k$

$r^{k+1} = r^k - \alpha_k A p^k$

Если $(|r^{k+1}|)$мало, остановиться.

Иначе:
$\beta_k = \frac{(r^{k+1}, r^{k+1})}{(r^k, r^k)}$
$p^{k+1} = r^{k+1} + \beta_k p^k.$

---

## Что важно в реализации

### 1. CG не обращает матрицу

Фраза $(A^{-1} b)$ в коде означает не явное обращение матрицы, а решение системы $(A x = b)$ итерационным способом.

### 2. Нужен правильный оператор (A v)

Для Пуассона обычно используется дискретный лапласиан на равномерной сетке.

### 3. Граничные условия должны быть встроены корректно

Если решается задача Дирихле, значения на границе надо фиксировать отдельно.

### 4. CG применим только к SPD-матрицам

Если матрица не симметрична или не положительно определена, нужен другой метод.

---

## Связь с методом Узавы

В методе Узавы на каждом шаге приходится решать уравнение вида
$A u^{k+1} = f - B^T p^k.$
Если (A) — дискретный лапласиан, то это и есть задача типа Пуассона.
Значит, CG часто используется как внутренний решатель внутри Uzawa.

---

## Пример кода

```python
import numpy as np


def apply_laplacian(u, h):
    """Возвращает -Δu на равномерной квадратной сетке с нулевыми границами."""
    N = u.shape[0]
    Lu = np.zeros_like(u)

    for i in range(1, N - 1):
        for j in range(1, N - 1):
            Lu[i, j] = (
                u[i + 1, j] + u[i - 1, j] +
                u[i, j + 1] + u[i, j - 1] -
                4.0 * u[i, j]
            ) / (h * h)

    return -Lu


def dot(u, v):
    return np.sum(u * v)


def cg_poisson(f, h, tol=1e-8, max_iter=1000):
    """Решает -Δu = f методом сопряжённых градиентов."""
    u = np.zeros_like(f)

    r = f - apply_laplacian(u, h)
    p = r.copy()
    rs_old = dot(r, r)

    if np.sqrt(rs_old) < tol:
        print("CG converged immediately")
        return u

    for k in range(max_iter):
        Ap = apply_laplacian(p, h)
        denom = dot(p, Ap)

        if abs(denom) < 1e-30:
            raise ZeroDivisionError("CG breakdown: denominator is too small")

        alpha = rs_old / denom
        u = u + alpha * p
        r = r - alpha * Ap

        rs_new = dot(r, r)
        if np.sqrt(rs_new) < tol:
            print(f"CG converged in {k + 1} iterations")
            return u

        beta = rs_new / rs_old
        p = r + beta * p
        rs_old = rs_new

    print("CG reached max_iter")
    return u
```

## Тестовый пример

Для проверки удобно взять правую часть, для которой известно точное решение.

Например, на квадрате $([0,1]\times[0,1])$:

$u(x,y) = \sin(\pi x)\sin(\pi y).$

Тогда $-\Delta u = 2\pi^2 \sin(\pi x)\sin(\pi y).$
Значит можно задать

```python
import numpy as np

N = 64
L = 1.0
h = L / (N - 1)

x = np.linspace(0, L, N)
y = np.linspace(0, L, N)
X, Y = np.meshgrid(x, y, indexing="ij")

u_exact = np.sin(np.pi * X) * np.sin(np.pi * Y)
f = 2.0 * np.pi**2 * u_exact

u_num = cg_poisson(f, h)

error = np.max(np.abs(u_num - u_exact))
print("max error =", error)
```

Этот тест хорош тем, что:

- решение известно аналитически;
    
- на границе оно равно нулю;
    
- можно проверить, правильно ли работает лапласиан и CG.
    

---

## Что стоит проверить при запуске

1. Невязка на старте не должна быть нулевой.
    
2. Число итераций не должно быть 0 для нетривиальной правой части.
    
3. Ошибка должна уменьшаться при увеличении числа узлов сетки.
    
4. Если заменить правую часть на случайную, CG должен работать дольше, чем на одной синусе.
    

---

## Связь с дальнейшим кодом для Узавы

Когда этот решатель заработает, его можно использовать как подпрограмму внутри шага Узавы:
$A u^{k+1} = f - B^T p^k.$ 
То есть основной внутренний блок метода Узавы — это именно такой решатель Пуассона.
